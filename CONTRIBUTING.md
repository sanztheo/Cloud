## Pull Request Process

### 1. Before you start

- Check if an Issue doesn't already exist
- Discuss major changes in an Issue first
- Read existing code to understand the style

### 2. Prepare your contribution

```bash
# Fork the repo on GitHub

# Clone your fork
git clone https://github.com/YOUR-USERNAME/cloud-browser.git

# Create a branch
git checkout -b feature/my-feature
# or
git checkout -b fix/my-bugfix
```

### 3. Code rules

- **Swift Style**: Follow Apple conventions
- **SwiftUI**: Use modifiers in standard order
- **Commits**: Clear messages in English or French
- **Tests**: Test your changes before submitting

### 4. Submit

```bash
git add .
git commit -m "feat: clear description of the change"
git push origin feature/my-feature
```

Then open a Pull Request on GitHub.

### 5. Review

- I review all PRs personally
- I may request changes
- I reserve the right to reject a PR without justification
- Accepted PRs will be merged by me only
